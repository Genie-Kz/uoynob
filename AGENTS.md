# AGENTS.md

## Git

- コミットは、1つの機能や修正単位で行うこと
- 修正が複数の機能や修正にまたがる場合は、コミットを分割すること
- コミットメッセージは、以下の点を意識して記述すること
  - 具体的であること
  - 詳細に変更内容を説明すること
  - 変更理由を明確にすること
  - コミットメッセージを1行のみで記述することは避け、後々見返したときに変更内容がわかるようにすること
- Co-Authored-byにエージェントを明示すること

## コーディング規約

### 全般

- コードの概要を把握できるように、日本語のコメントを多めに記述する
- コメントは冗長でも構わないので、コードの意図や処理の流れを明確にする
- 条件分岐・ループ・例外処理の直前にはその意図を説明するコメントを追加する
- コードの可読性を高めるため、適切な変数名や関数名を使用する
- コードの再利用性を高めるため、共通化できる処理は関数にまとめる。ただし、過度な抽象化は避けること

### TypeScript/Vue

- コードの編集後はESLint によるコードリンターと TypeScript による型チェッカーを実行すること
- TypeScriptによる型安全性を確保する
- ビューとビジネスロジックが密結合しないよう、純粋関数を意識してコードを書く

### CSSスタイリング

- 新規に UI を実装する際は、すでに実装されている他のコンポーネントやページの大まかなデザインの方向性を踏襲すること
