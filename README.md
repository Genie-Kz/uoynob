# 凡庸な イルルカSP（クライアントサイド再現版）

今宵もみんなをいざなっちゃうよ～！
これは、サービス終了した攻略サイト「**凡庸な イルルカSP**」（`bonyou.info/dqm2sp`）を、
**GitHub Pages で動く静的サイト**として再現したものなのです。
元サイトがサーバーサイドで動かしていた処理（モンスター検索など）は、ぜ～んぶ**ブラウザ内（クライアントサイド）**で動くようにしてあるよ♪

対象ゲーム：**ドラゴンクエストモンスターズ２ イルとルカの不思議な鍵SP**

---

## できること（実装済み機能）いざな～っちゃうよ！

| ページ | 内容 |
|---|---|
| `/`（トップ） | サイト紹介・カテゴリナビ・シェアボタン |
| `/monster/` | モンスター図鑑（一覧）。`?rank=` `?lineage=` `?size=` `?q=` で絞り込み。DataTablesで検索・ソート・ページング |
| `/monster/detail/?id=001-1` | モンスター詳細。特性・耐性30種（色分け）・ステータス・装備 |
| `/search-monster/` | **モンスター検索**。耐性30種の閾値（◯◯↑）＋特性で絞り込み。元サイトのAPIを完全にJS化 |
| `/simulator-monster-build/` | **ビルドシミュレーター（モンスター選択）**。検索→一覧から選ぶとビルド画面へ |
| `/simulator-monster-build/build/?id=001-1` | **ビルド画面**。特性・スキル・武器鍛冶を編集モーダルで入れ替えて最終耐性を計算 |
| `/skill/` | スキル一覧（425件）。`?cat=ability`（特技系）／`?cat=parameter`（パラメータ系）で絞り込み |
| `/skill/detail/?id=001` | スキル詳細。スキル構成（ガード＋に耐性+2バッジ）・このスキルを持つモンスター |

- モンスター詳細ページには、そのモンスターが持つ**スキル**欄も表示（種族→スキルの逆引き）。**モンスター名を押す**とビルド画面へ移動します。
- ビルド画面では、ボディサイズに応じて**特性枠**（6/6/7/8/9）と**スキル枠**（3/3/4/5/6）が変化。特性は既定値を初期表示、各枠の「編集」ボタンで開くモーダル（検索input＋青枠の一覧）から入れ替え。スキルは初期空欄、武器鍛冶は3枠。スキルは**全て習得済み（ガード＋発動）前提**で計算します。
- ナビのアコーディオンは各項目を縦1行ずつ表示、耐性表は「耐性名（グレー背景）＝上／値（色付き）＝下」の3列レイアウト（いずれも元サイト準拠）。

### ビルドシミュレーターの計算仕様（日々の努力の玉藻の前というか）
耐性段階：弱点／普通／軽減／半減／激減／無効／回復(反射)。内部では 弱点=0 … 反射=7 で計算するよ。
- **ボディサイズ**：メガ+2／ギガ+3／超ギガ+4（ザキ・毒・呪い・混乱・マインド・マヒ・眠り）。図鑑値は「元サイズの状態」なので、**元サイズからの差分**で適用（二重計上を防止）。枠数も連動（スタンダード/スモール6・メガ7・ギガ8・超ギガ9）。
- **全ガード＋**：全耐性+1。**メタル系**：毒・呪い・混乱・マヒ・眠り+1（ライトメタルは耐性影響なし）。**こうどう**：おそい+2／はやい-2／超はやい-4（毒・呪い・マインド・混乱・マヒ・眠り）。メタル系・こうどう系は各1つまで。
- **スキル**：`〇〇ガード＋` 1つにつき該当耐性+2（重複可。ご当地スキルの+4も再現）。
- **武器鍛冶**：別々の耐性を3つまで各+1。
- **上限・下限**：スキル等での上昇は**回復どまり**（反射にはならない＝反射は元々のみ）。下降で弱点を下回ると**内部的に負の蓄積**となり、それを上回るまで弱点のまま。

## データについて（日々の努力の玉藻の前というか）

- 全モンスターのデータは `data/monsters.json`（908体・56項目）。
- 元データ（`MonsterData.json`）から `build-data.js` で生成してるのです。
- スキルデータは `data/skills.json`（425件）。Web Archive から `scrape-skills.js` で取得・生成してるよ。

### データを更新したいとき

```bash
node build-data.js "path/to/MonsterData.json"   # モンスター
node scrape-skills.js                            # スキル（Web Archiveから取得）
```

`data/monsters.json` が再生成されるよ。各モンスターには `id`（例 `001-1` ＝ 位階3桁-連番）、`no`、`base`（本体名）、`personality`（性格）を付与してるのです。

## ローカルで動作確認

```bash
node serve.js 8123
# → http://localhost:8123/ をブラウザで開く
```

`file://` で直接開くと `fetch` でデータを読めないので、必ずローカルサーバー経由で見てね（ごめん、ドジっちゃった…にならないように）。

## GitHub Pages で公開

1. このフォルダの中身をリポジトリのルートに置いてプッシュ
2. リポジトリの **Settings → Pages** で、公開ブランチ（例 `main` / `/root`）を指定
3. 数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開！

- 相対パス＋各ページの `window.SITE_ROOT` で組んでるので、サブパス公開でもそのまま動くよ。
- `.nojekyll` を置いてあるので Jekyll の処理は行われないのです。
